# 🎯 EC2 네트워크 설정 빠른 가이드

## ✅ 네트워크 설정 요약

EC2 인스턴스 생성 시 네트워크 설정 화면에서:

### 1. VPC (Virtual Private Cloud)
```
✅ 기본 VPC 선택 (자동)
```
- **설정**: 그대로 두기
- **이유**: AWS가 자동으로 생성한 기본 VPC 사용

---

### 2. 서브넷 (Subnet)
```
✅ 기본 서브넷 선택 (자동)
```
- **설정**: 그대로 두기
- **이유**: 퍼블릭 서브넷이 자동 선택됨

---

### 3. 가용 영역 (Availability Zone)
```
✅ 자동 선택 (권장)
```
- **설정**: 그대로 두기
- **이유**: AWS가 최적의 가용 영역 선택

---

### 4. 퍼블릭 IP 자동 할당 ⚠️ **중요!**
```
⚠️ 반드시 "활성화"로 설정!
```
- **설정**: **활성화** 선택
- **이유**: 이것만 활성화하지 않으면 외부에서 접속 불가
- **주의**: 기본값이 "비활성화"일 수 있음

---

### 5. 방화벽 (보안 그룹)
```
✅ 새 보안 그룹 생성
```
- **설정**: 아래 규칙 추가

#### 필수 보안 그룹 규칙

| 유형 | 포트 | 소스 | 용도 |
|------|------|------|------|
| SSH | 22 | 내 IP | SSH 접속 |
| HTTP | 80 | 0.0.0.0/0 | 웹 접속 |
| HTTPS | 443 | 0.0.0.0/0 | HTTPS 접속 |
| Custom TCP | 8000 | 0.0.0.0/0 | FastAPI |

### EC2 인스턴스 시작 화면

```
┌─────────────────────────────────────────────────┐
│ 네트워크 설정                                    │
├─────────────────────────────────────────────────┤
│                                                 │
│ VPC - 필수                                      │
│ ▼ vpc-xxxxx (기본)              ← 그대로 두기   │
│                                                 │
│ 서브넷                                          │
│ ▼ 기본 서브넷 선택               ← 그대로 두기   │
│                                                 │
│ 퍼블릭 IP 자동 할당                             │
│ ● 활성화  ○ 비활성화            ← 활성화 선택!  │
│                                                 │
│ 방화벽 (보안 그룹)                              │
│ ● 보안 그룹 생성                ← 새로 생성     │
│ ○ 기존 보안 그룹 선택                           │
│                                                 │
│ 보안 그룹 규칙                                  │
│ [+] 보안 그룹 규칙 추가         ← 규칙 추가     │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 📝 체크리스트

배포 전 네트워크 설정 확인:

- [ ] VPC: 기본 VPC 선택됨
- [ ] 서브넷: 기본 서브넷 선택됨
- [ ] 가용 영역: 자동 선택
- [ ] **퍼블릭 IP 자동 할당: 활성화** ⚠️
- [ ] 보안 그룹: 새로 생성
- [ ] SSH (22) 규칙 추가
- [ ] HTTP (80) 규칙 추가
- [ ] HTTPS (443) 규칙 추가
- [ ] Custom TCP (8000) 규칙 추가

## ⚠️ 주의사항

### 1. 퍼블릭 IP 자동 할당
- **가장 중요한 설정입니다!**
- 비활성화 시 외부 접속 불가
- 나중에 변경하려면 Elastic IP 설정 필요

### 2. 보안 그룹
- SSH 포트는 "내 IP"로 제한 권장
- 테스트 시에만 0.0.0.0/0 사용
- 배포 후 불필요한 포트 닫기

### 3. VPC/서브넷
- 기본 설정으로 충분
- 고급 설정은 나중에 변경 가능

## 🔧 배포 후 확인

### 1. 퍼블릭 IP 확인
```bash
# EC2 콘솔에서 확인
인스턴스 선택 → 세부 정보 → 퍼블릭 IPv4 주소
```

### 2. 보안 그룹 확인
```bash
# EC2 콘솔에서 확인
인스턴스 선택 → 보안 → 보안 그룹
```

### 3. 접속 테스트
```bash
# SSH 접속
ssh -i key.pem ubuntu@YOUR_PUBLIC_IP

# 웹 접속 (배포 후)
http://YOUR_PUBLIC_IP
```

## 🆘 문제 해결

### 문제: SSH 접속 안 됨
```
원인: 보안 그룹에 SSH 규칙 없음
해결: 보안 그룹에 포트 22 규칙 추가
```

### 문제: 웹사이트 접속 안 됨
```
원인 1: 퍼블릭 IP 자동 할당 비활성화
해결: Elastic IP 할당

원인 2: 보안 그룹에 HTTP 규칙 없음
해결: 보안 그룹에 포트 80 규칙 추가
```

### 문제: API 호출 실패
```
원인: 보안 그룹에 포트 8000 규칙 없음
해결: 보안 그룹에 Custom TCP 8000 규칙 추가
```

## 💡 추가 팁

### Elastic IP (고정 IP)
인스턴스를 중지했다 시작하면 퍼블릭 IP가 변경됩니다.
고정 IP가 필요하면:

```bash
1. EC2 콘솔 → Elastic IP
2. "Elastic IP 주소 할당"
3. 할당된 IP를 인스턴스에 연결
```

### 보안 그룹 수정
배포 후에도 보안 그룹 규칙 수정 가능:

```bash
1. EC2 콘솔 → 보안 그룹
2. 해당 보안 그룹 선택
3. "인바운드 규칙 편집"
```
