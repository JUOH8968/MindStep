# 📘 페르소나 넛지(Nudge) 시스템 최종 명세서 (v6.0)

**작성일:** 2026. 01. 12
**상태:** ✅ 최종 완료 (Production Ready)

---

## 1. 시스템 개요
사용자의 성향(MBTI 등)에 맞춰 **가장 효과적인 말투와 타이밍**으로 할 일(Task) 수행을 독려하는 지능형 AI 코칭 시스템입니다. 
단순 알림을 넘어, 사용자의 **감정 상태**와 **작업 진척도**를 고려해 능동적으로 말을 겁니다.

---

## 2. 핵심 기능 요약

### 🎯 1) 스마트 타겟팅 (Smart Targeting)
- **'시작' 상태 집중:** 수많은 할 일 중 사용자가 **'시작' 버튼을 누른 Task**만 콕 집어 코칭합니다.
- **우선순위 자동 정렬:** 여러 일을 벌려놓아도, **마감 시간이 가장 급한 일**부터 먼저 처리하도록 유도합니다.
- **이미 끝난 일 무시:** '완료'된 Task는 넛지 대상에서 즉시 제외되어, 뒷북치는 실수를 하지 않습니다.

### ⚡ 2) 초고속 반응 (Instant Reaction)
- **1초 딜레이:** 사용자가 '시작' 버튼을 누르면, **단 1초 만에** 첫 번째 응원/지시 메시지가 도착합니다.
(기존 5~10초 지연을 획기적으로 단축하여 답답함을 없앴습니다.)

### 🔄 3) 완벽한 재시작 (Seamless Restart)
- **자동 감지:** 모든 일을 끝내고 넛지가 멈췄더라도, **새로운 일을 추가하고 '시작'하면 즉시 시스템이 깨어납니다.**
- 백엔드 상태 관리 로직이 개선되어, 별도의 새로고침 없이 연속적인 코칭이 가능합니다.

---

## 3. 기술 명세 (Technical Specs)

### 🧠 AI 모델: `gemini-2.0-flash`
- **선정 이유:** 초기 실험 모델(2.0-flash-exp)의 불안정성과 구버전(1.5-flash)의 호환성 문제를 모두 해결한 **최신 안정화 버전**입니다.
- **장점:** 빠른 응답 속도, 높은 API 허용량(Quota), 안정적인 24시간 가동 능력.

### ⚙️ 최적화 전략 (Optimization)
1. **타이머/폴링 구조:**
   - **Start:** 시작 직후 1초 뒤 첫 폴링 (즉각 반응)
   - **Ongoing:** 이후 설정된 간격(테스트 시 5~10초)으로 폴링
   - **Stop:** 일시정지/완료 시 타이머 즉시 해제 (리소스 절약)

2. **비용 효율화:**
   - **심야 자동 절전:** 23:00 ~ 05:00 사이에는 API 호출을 원천 차단 (무료)
   - **이중 필터링:** 프론트엔드와 백엔드 양쪽에서 '시작 중인 Task'가 없으면 호출을 막습니다.

## 4. 페르소나 4인방 (Character Logic)

사용자의 성향에 따라 아래 4가지 유형 중 하나가 자동 배정됩니다.

| 타입 | 이름 | 역할 | 넛지 스타일 |
|---|---|---|---|
| **Type A** | **Lighthouse**<br>(온화한 등대) | 지친 영혼의 쉼터 | "조금만 더 힘내볼까요? 힘들면 물 한 잔 마셔요."<br>(공감 & 휴식 권유) |
| **Type B** | **DrillSergeantFocus**<br>(단호한 교관 1) | 산만한 주의력 교정 | "딴생각 금지! 지금 당장 10분만 집중한다. 실시!"<br>(짧고 강력한 명령) |
| **Type C** | **DrillSergeantPace**<br>(단호한 교관 2) | 오버페이스 조절 | "속도보다 완주가 중요하다. 천천히 끝까지 간다."<br>(지속 가능성 강조) |
| **Type D** | **Strategist**<br>(냉철한 전략가) | 시작 못하는 완벽주의 | "데이터상 지금 시작해야 98% 확률로 끝냅니다."<br>(이성적/수치적 설득) |

## 5. 트러블슈팅 이력 (Solved Issues)

| 문제 현상 | 원인 | 해결책 |
|---|---|---|
| **API 429 에러** | 너무 잦은 호출로 인한 차단 | 모델을 고성능(`gemini-2.0-flash`)으로 교체 |
| **API 404 에러** | 잘못된 모델명(`1.5-flash`) 사용 | 검증된 모델명(`2.0-flash`)으로 수정 |
| **재시작 안 됨** | 모든 Task 완료 시 상태가 'Off'로 고정됨 | 새 Task 시작 시 상태를 강제 'On'하는 로직 추가 |
| **완료된 일 언급** | 대화 기록에 남은 옛날 정보를 참조함 | 프롬프트에서 대화 기록 삭제 & '시작' Task만 주입 |

## 6. 사용 가이드

1. **할 일 등록:** 우측 패널에서 할 일을 입력하고 추가하세요. (이때는 넛지 안 옴)
2. **시작(Start):** 할 일 옆의 `▶` 버튼을 누르세요. **1초 뒤** 코치가 말을 겁니다.
3. **일시정지(Pause):** 잠깐 쉴 때는 `⏸` 버튼을 누르세요. 넛지가 멈춥니다.
4. **완료(Check):** 다 했으면 체크박스를 누르세요. "수고하셨습니다!" 인사를 받습니다.
