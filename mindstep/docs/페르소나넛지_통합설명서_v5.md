# 📘 페르소나 넛지(Nudge) 기능 통합 설명서

**버전:** v5.0 Final
**작성일:** 2026. 01. 12

---

## 1. 개요
**페르소나 넛지**는 사용자가 등록한 할 일(Task)을 미루거나 잊지 않도록, AI 코치가 페르소나에 맞춰 능동적으로 말을 거는 시스템입니다. 
단순한 알림이 아니라, 사용자의 성향(Type A~D)에 맞춘 **말투, 간격, 접근 방식**으로 동기를 부여합니다.

---

## 2. 페르소나 4가지 유형

사용자의 성향에 따라 4명의 코치 중 한 명이 배정됩니다.

| 타입 | 페르소나 이름 | 특징 | 넛지 스타일 | 간격 (테스트/실제) |
|---|---|---|---|---|
| **Type A** | **Lighthouse**<br/>(온화한 등대) | 공감, 수용, 자기자비 | "힘드시면 쉬었다 하세요.", "물 한 잔 어때요?"<br/>비난 없이 부드럽게 격려 | 10초 / 75분 |
| **Type B** | **DrillSergeantFocus**<br/>(단호한 교관 - 집중) | 명령조, 몰입 강조 | "딴짓 하지 마라!", "지금 당장 시작해!"<br/>짧고 강렬한 지시 | 5초 / 7.5분 |
| **Type C** | **DrillSergeantPace**<br/>(단호한 교관 - 완주) | 페이스 조절, 완주 목표 | "속도보다 완주다.", "지치지 말고 끝까지!"<br/>지속 가능한 실행 독려 | 7초 / 25분 |
| **Type D** | **Strategist**<br/>(냉철한 전략가) | 논리, 데이터, 효율 | "데이터 분석 결과...", "80% 완성도로 끝내시죠."<br/>이성적이고 분석적인 제안 | 8초 / 45분 |

## 3. 핵심 작동 로직 (최종 수정 반영)

### ✅ 넛지 발생 조건 (AND 조건)
다음 조건이 **모두** 충족될 때만 넛지 메시지가 생성됩니다.

1. **Task 상태:** **'시작'** 상태인 Task가 최소 1개 이상 있을 때
   - *'대기', '일시정지', '미루기', '포기', '완료' 상태는 넛지 대상이 아닙니다.*
2. **시간:** 방해 금지 시간(23:00 ~ 05:00)이 아닐 때
3. **간격:** 페르소나별 설정된 시간이 지났을 때

### 🔄 우선순위 정렬
여러 개의 Task가 '시작' 상태일 경우, **가장 이른 시간(날짜+시간)**의 Task를 최우선으로 언급합니다.
- 예: 14:00 Task와 16:00 Task가 모두 시작 중이면, 14:00 Task에 대해 넛지를 보냅니다.

### 🕹️ 사용자 행동에 따른 반응

| 사용자 행동 | 넛지 시스템 반응 | 비고 |
|---|---|---|
| **Task 추가** | 넛지 대기 (타이머 시작 X) | '대기' 상태이므로 넛지 안 함 |
| **'시작' 클릭** | **넛지 타이머 시작 🟢** | 5~10초 후 첫 메시지 도착 |
| **'일시정지' 클릭** | 넛지 타이머 중지 🔴 | 더 이상 메시지 안 옴 |
| **'미루기/포기' 클릭** | 넛지 타이머 중지 🔴 | |
| **'완료' 클릭** | 해당 Task 제외 | 남은 '시작' Task가 없으면 중지 |
| **모두 완료** | **축하 메시지 + 자동 종료** 🎉 | "수고하셨습니다!" |
| **새 Task 추가** | 시스템 재활성화 대기 | 이후 '시작' 누르면 다시 작동 |

### 🧠 스마트 키워드 감지
사용자가 채팅창에 **"다했어", "끝", "완료"** 등을 입력했을 때:
- **남은 Task 확인:** 아직 완료되지 않은 Task가 있는지 확인합니다.
- **Task 0개:** "수고하셨습니다!" 메시지와 함께 넛지 완전 종료.
- **Task 1개 이상:** 넛지를 종료하지 않고 **계속 유지**합니다. (실수 방지)

## 4. 시스템 구조 및 비용 절감

### 작동 방식
1. **프론트엔드:** 설정된 간격마다 서버에 "넛지 필요해?"라고 질문 (/api/nudge/poll)
2. **백엔드:**
   - 23시~05시인가? (무료 체크) → ❌ 거절
   - '시작' 중인 Task가 있는가? (DB 조회) → 없으면 ❌ 거절
   - 있다면 **Gemini API** 호출 (유료) → AI 페르소나 넛지 메시지 생성
3. **결과:** 생성된 메시지를 채팅창에 표시

### 비용 최적화 (80~90% 절감)
- **방해 금지 시간:** 심야 시간대 API 호출 원천 차단
- **상태 필터링:** '시작' 상태가 없으면 API 호출 안 함
- **완료 Task 제외:** 이미 끝난 일로 API 낭비 방지
- **프로덕션 간격:** 실제 배포 시 간격을 분 단위(7.5분~75분)로 늘려 비용 최소화

## 5. 자주 묻는 질문 (FAQ)

**Q. Task를 추가했는데 왜 넛지가 안 오나요?**
A. Task를 추가하면 '대기' 상태가 됩니다. **'시작' 버튼**을 눌러야 코칭이 시작됩니다. 준비가 되면 시작을 눌러주세요!

**Q. 잠깐 쉬고 싶은데 넛지가 계속 와요.**
A. Task의 **'일시정지'** 버튼을 눌러주세요. 넛지가 멈춥니다. 다시 할 때 '시작'을 누르면 됩니다.

**Q. "다했다"고 말했는데 넛지가 계속 나와요.**
A. 아직 완료 처리가 안 된 Task가 남아있기 때문입니다. 시스템이 사용자를 끝까지 챙기기 위해, 남은 일이 있으면 넛지를 멈추지 않습니다.

**Q. 밤에는 알림을 받고 싶지 않아요.**
A. 밤 11시(23:00)부터 아침 5시(05:00)까지는 **방해 금지 시간**으로, 자동으로 넛지가 오지 않습니다.
