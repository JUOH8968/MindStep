## 1. 시간 체크 (방해 금지 시간)

### 💰 비용
**완전 무료** - 서버/클라이언트의 시스템 시간 확인만 사용

### ⏱️ 소요 시간
**거의 0ms** - 즉각적 (밀리초 단위)

### 🔧 작동 원리

#### 프론트엔드 체크 (JavaScript)
```javascript
const isDoNotDisturbTime = () => {
    const hour = new Date().getHours(); // 현재 시간 확인
    return hour >= 23 || hour < 5;      // 23시~5시 체크
};
```

**실행 위치:** `PersonaCoachApp.js` - `pollNudge()` 함수 내부

**체크 시점:** API 호출 전

**효과:** 불필요한 네트워크 요청 방지

#### 백엔드 체크 (Python)
```python
def is_do_not_disturb_time():
    current_hour = datetime.now().hour  # 현재 시간 확인
    if current_hour >= 23 or current_hour < 5:
        return True
    return False
```

**실행 위치:** `gemini_service.py` - `generate_nudge_message()` 함수 내부

**체크 시점:** 넛지 메시지 생성 전

**효과:** Gemini API 호출 방지 → 비용 절감

### 📊 이중 체크 시스템

```
사용자 브라우저
    ↓
[1차 체크] 프론트엔드 (23:00~05:00 확인)
    ↓ (통과 시에만 API 호출)
백엔드 서버
    ↓
[2차 체크] 백엔드 (23:00~05:00 재확인)
    ↓ (통과 시에만 Gemini API 호출)
Gemini API
```

**장점:**
- 네트워크 통신 없음
- CPU 사용량 극히 미미
- 이중 안전장치로 완벽한 차단

**비용 절감 효과:**
- 하루 6시간 (23:00~05:00) = **25% 시간**
- Gemini API 호출 **25% 감소**
- 월간 비용 약 **$10~$20 절감** (사용자 100명 기준)

## 3. 프론트엔드 폴링

### 💰 비용
**서버 비용만 발생** (매우 저렴)
- 서버 CPU/메모리 사용
- 데이터베이스 쿼리
- 네트워크 대역폭

**예상 서버 비용:**
- 사용자 100명: 월 $5~$10
- 사용자 1,000명: 월 $20~$50

### ⏱️ 폴링 간격

#### 테스트 환경
```javascript
const TEST_INTERVALS = {
    'Lighthouse': 10000,           // 10초
    'DrillSergeantFocus': 5000,    // 5초
    'DrillSergeantPace': 7000,     // 7초
    'Strategist': 8000             // 8초
};
```

#### 프로덕션 환경
```javascript
const PRODUCTION_INTERVALS = {
    'Lighthouse': 75 * 60 * 1000,          // 75분
    'DrillSergeantFocus': 7.5 * 60 * 1000, // 7.5분
    'DrillSergeantPace': 25 * 60 * 1000,   // 25분
    'Strategist': 45 * 60 * 1000           // 45분
};
```

### ⏱️ API 요청 응답 시간

```
DB 조회 (User, Task, Chat) → 10~50ms
    ↓
넛지 생성 로직 → 5~10ms
    ↓
Gemini API 호출 → 1~3초
    ↓
응답 전송 → 10~50ms

총 소요 시간: 1~3.5초
```

### 🔧 작동 원리

#### 타이머 시작
```javascript
const startNudgeTimer = () => {
    // 기존 타이머 제거 (중복 방지)
    if (nudgeTimerRef.current) {
        clearInterval(nudgeTimerRef.current);
        nudgeTimerRef.current = null;
    }

    // 페르소나별 간격 설정
    const interval = TEST_INTERVALS[settings.persona_type];
    console.log(`넛지 타이머 시작 - ${settings.persona_type}: ${interval/1000}초 간격`);

    // 첫 넛지 (간격의 절반 후 - 빠른 초기 반응)
    setTimeout(pollNudge, interval / 2);

    // 반복 넛지 (설정된 간격마다)
    nudgeTimerRef.current = setInterval(pollNudge, interval);
};
```

#### 넛지 폴링 함수
```javascript
const pollNudge = async () => {
    // 1. 로컬 체크: 방해 금지 시간
    if (!nudgeEnabled || isDoNotDisturbTime()) {
        return; // API 호출하지 않음
    }

    // 2. API 호출
    try {
        const result = await api("/api/nudge/poll");
        
        // 3. 메시지가 있으면 채팅창에 추가
        if (result.nudge_message && result.enabled) {
            setChatLogs(prev => [...prev, {
                id: Date.now(),
                sender: 'AI',
                content: result.nudge_message,
                time: new Date().toLocaleTimeString([...]),
                persona: result.persona_type
            }]);
        }
    } catch (err) {
        console.error("넛지 폴링 실패:", err);
    }
};
```

#### 흐름도
```
앱 로딩 완료
    ↓
startNudgeTimer() 실행
    ↓
간격/2 대기 (Type B: 2.5초, Type A: 5초)
    ↓
pollNudge() 실행
    ↓
    ├─ 방해 금지 시간? → return (비용 0)
    ├─ nudgeEnabled=false? → return
    └─ API 호출
        ↓
    백엔드 처리 + Gemini API
        ↓
    메시지 수신
        ↓
    채팅창 표시
        ↓
설정 간격 대기 (Type B: 5초, Type A: 10초)
    ↓
pollNudge() 재실행 (반복)
```

### 🔄 타이머 재시작 트리거

1. ✅ **초기 로딩 완료**
2. ✅ **Task 상태 변경** (시작, 일시정지, 미루기, 포기)
3. ✅ **채팅 메시지 전송** (키워드 감지 제외)
4. ✅ **새 Task 추가**

### 🛑 타이머 중지 트리거

1. ✅ **모든 Task 완료**
2. ✅ **키워드 감지 + 남은 Task 0개**
3. ✅ **nudgeEnabled = false**
